# Безопасность и сообщения об ошибках

Для обычных ошибок используйте раздел **Issues** репозитория.

Перед публикацией сообщения:

- удалите ФИО, адреса, реквизиты и иные избыточные данные из скриншотов;
- не прикладывайте cookies, токены, пароли и полный HAR;
- не публикуйте материалы ограниченного доступа;
- укажите версию Chrome, версию расширения и последовательность действий;
- по возможности воспроизведите ошибку на общедоступном деле.

Если описание само по себе раскрывает чувствительные данные или уязвимость, не создавайте публичный Issue. Используйте приватный канал, указанный в профиле безопасности репозитория или на странице расширения в Chrome Web Store.

Разработчик не запрашивает данные авторизации и не нуждается в доступе к пользовательскому профилю Chrome для диагностики.
